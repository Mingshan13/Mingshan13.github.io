方法重写

发表于 2020-03-17 更新于 2020-03-18 分类于 入门学习

方法重写：子类中出现了和父类方法中方法声明一模一样的方法。一模一样包括方法名和参数列表以及返回值类型

方法名相同时，使用的是子类自己的方法

为什么需要方法重写：

子类需要父类的功能，而子类又需要自己特有的内容时，需要重写父类里的方法

在子类的方法重写里，用super（。。。）调用父类

方法重写的注意事项

  • 父类中的私有方法不能重写，子类根本就不能继承父类的私有，更别说重写了

  • 子类重写父类方法时，访问权限不能更低

    父类是public，子类也得是public。子类的权限可以比父类高，一样，但是不能低

  • 父类静态方法，子类也必须静态方法重写

继承中构造方法的关系

发表于 2020-03-17 更新于 2020-03-18 分类于 入门学习

子类中 所有的构造方法 默认都会访问父类中空参数的构造方法

子类中所有的构造方法，不管是有参还是无参

因为继承关系，子类可能会继承和使用父类中的数据，所以在子类初始化之前，父类要先初始化

注意：子类的每一个构造方法的第一条语句默认都是super（）；，没写也默认有

如果父类没有无参构造方法

——会报错

如果父类里什么构造方法都没有，则默认产生一个无参构造方法。此时不会报错

如果父类里有有参构造方法，无参构造方法就不自己生成了，这时候又不自己给无参构造方法，就会报错。

解决方法：1. 在父类中加一个无参

​ 2. 子类通过super去显示调用父类其他的带参的构造方法

​ 3. 子类通过this去调用本类的其他构造方法

​ 用这个方法，子类中一定要有一个访问了父类的构造方法，否则父类数据就没有初始化

关于super显式与隐式的思考

为什么父类中没有无参构造函数就会报错，我在这里可否认为是因为每一个子类的构造方法里的第一行都有一个默认的super（），即使我没有写这一句，它也是隐式存在的

所以调用子类构造方法的时候，首先就执行super（），即父类的空构造方法，如果父类没有空构造方法就报错

为了解决这个问题，可以更改默认的super（），显式的把super写出来，并且是super（…）的形式，不调用父类的空构造方法，而是去调用已有的有参构造方法，这就破解掉了默认的super（）总是要调用父类空构造方法的问题了

换而言之，如果子类的构造方法第一句没有写super（…)，也能运行成功，说明父类一定有空构造方法

关于子类通过this调用本类的其他方法的思考

this是代表本类的，this（）代表本类的空构造方法，如果在子类的某个带参构造方法里的第一行写这个，就去找本类的空构造方法，而这个空构造方法里一定要能访问父类的构造方法

意思就是，我在本类的带参构造方法里用了this（），本类的空构造方法里一定要有this（…)访问到父类的带参构造方法，不然还是会报错

这里为什么使用了this（）就不再有默认的super（）尚且存疑

另外，不管是super（。。。）还是this（。。。）必须出现在第一条语句上，如果不放在第一条语句，就可能对父类进行了多次初始化

this关键字、super关键字

一、this关键字

  • this代表所在类的对象引用，它就是当前类的对象名，就代表当前对象

    因为在类里面不能用class.的方式调用成员变量，所以引用this来调用类里面的成员变量

  • 方法被哪个对象调用，this就代表哪个对象

  • this应用场景：

    • 解决局部变量隐藏成员变量

    针对调用父类里的东西，引入super关键字

二、super关键字

super代表父类存储空间的标识，也就是父类的引用操作父类的成员

三、this与super

  1. 调用成员变量

this.成员变量 调用本类的成员变量

super.成员变量 调用父类的成员变量

  1. 调用构造方法

this(…) 调用本类的构造方法

super(…) 调用父类的构造方法

因为继承只能继承父类的成员变量和成员方法，不能继承父类的构造方法，这里就用super来调用父类的构造方法

  1. 调用成员方法

this.成员方法 调用本类的成员方法

super.成员方法 调用父类的成员方法

main方法的格式解释

发表于 2020-03-15 分类于 入门学习

public static void main(String []args){ }

public: 公共的，访问权限是最大的，因为main被jvm（虚拟机）调用，所以权限要是最大的

static: 静态的，不需要创建对象，方便jvm直接通过类名调用

void: 方法的返回值是返回给调用者，main的调用者是jvm，给jvm返回值没有意义

main: 一个常见的方法入口，几乎所有语言都是以main作为入口

String[]args: 一个空字符串数组，早期用来接收键盘数据，后来有了scanner就不怎么用了，但是有固定的输入格式

final的特点

final：常用于修饰类、方法、变量

  1. 修饰类的特点：如果一个类被final修饰，这个类叫最终类，最终类无法被继承

  2. 修饰方法的特点：被final修饰的方法不能被重写

虽然是静态的，但是是可改变的，只要不再次被改变，当前的值就一直用下去

二、特点

1.它可以修饰成员变量，也可以修饰成员方法

2.随着类的加载而加载

3.优先于对象存在（对象是在new的时候才存在，而class文件加载的时候还没有创建对象）

4.被类的所有对象共享，比如一个学校的学生共用一个学校名字

5.可以通过类名调用，本身也可以通过对象名调用。推荐使用类名调用，静态修饰的内容一般都是与类相关的类成员，非静态的是对象成员

main方法也是静态的，因为它是被虚拟机调用，虚拟机调用main的时候就不用创建对象，直接类名.main

三、注意事项

1.在静态方法中没有this关键字

因为静态是随着类的加载而加载，this是随着对象的创建而存在的。静态先出现，this后出现，先有的东西去访问后面才有的现在还没有的东西，肯定会出问题，注意先后顺序！

2.静态方法只能访问静态的成员变量和静态的成员方法

非静态方法访问的成员变量和成员方法可以使静态的也可以是非静态的

四、静态变量和成员变量的区别

1.所属不同

静态变量属于类，是类变量。成员变量属于对象，是示例变量，对象变量

2.内存位置不同

静态变量存储在方法区的静态区，成员变量存储在堆内存

3.内存出现时间不同

静态变量随着类的加载而加载，随着类的消失而消失

成员变量随着对象的创建而存在，随着对象的消失而消失

4.调用不同

静态变量可以通过类名调用，也可以通过对象名调用

成员变量只能通过对象名调用
