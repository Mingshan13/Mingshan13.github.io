二、html颜色

1.预定义/跨浏览器颜色名

所有浏览器都支持颜色名

2.十六进制色

所有浏览器都支持

#RRGGBB

RR为红色成分，GG为绿色成分，BB为蓝色成分

所有值都介于0到ff之间，ff最大

#ff0000是红色

#00ff00是绿色

#0000ff是蓝色

3.RGB颜色

所有浏览器都支持

rgb（red,green,blue)每个参数是对应颜色的强度介于0到255

4.RGBA颜色（透明色）

RGBA是RGB的拓展，带有alpha通道（就是多出来的那个A），规定不透明度

alpha参数介于0.0（完全透明）与1.0（完全不透明）之间

透明应用的延伸

加了transparent（全透明）以后，就变成透明的了，但是选中后依然可见

接口

发表于 2020-03-20 分类于 入门学习

一、接口特点

  1. 接口用关键字interface表示

    interface 接口名 {}

  2. 类实现接口用implements表示

    class 类名 implements 接口名{}

  3. 接口不能实例化，按照多态的方式实例化

  4. 接口的子类

    1. 可以是抽象类，但意义不大（因为抽象类也不能实例化，还要再创建子类，没什么用
    2. 可以是具体类，但要重写接口中的所有抽象方法（用这个！）

由此可见，多态有这样几种方式

  1. 具体类多态（几乎不用）
  2. 抽象类多态（常用）
  3. 接口多态（最常用）

二、接口成员特点

  1. 成员变量：只能是常量的，并且是静态的（可以通过接口名.变量名访问）

    ​ 默认修饰符：public static final

  2. 构造方法：接口没有构造方法

    ​ 所有的类都默认继承自Object类

  3. 成员方法：只能是抽象方法

    ​ 默认修饰符：public abstract

抽象类

发表于 2020-03-20 分类于 入门学习

在Java中，一个没有方法体的方法应该定义为抽象方法，如果类中有抽象方法，该类必须定义为抽象类

注意什么叫“没有方法体”

上面的只有圆括号，没有花括号的叫没有方法体

下面这种叫空方法体，与没有方法体不一样

一、抽象类的特点

  1. 抽象类和抽象方法必须用abstract关键字修饰

  2. 抽象类中不一定有抽象方法，但是有抽象方法的类必须定义为抽象类

  3. 抽象类不能示例化，因为它不是具体的

    抽象类有构造方法，但是不能实例化，这个构造方法的作用是用于子类访问父类数据的初始化

  4. 抽象类的子类

    1. 如果不想重写抽象方法，抽象类的子类是只能是抽象子类

    2. 如果重写所有的抽象方法，抽象类的子类可以使具体的类

  5. 抽象类的实例化其实是靠具体的子类实现的，使用了多态的方式

    多态大多数时候是与抽象类结合起来用的，多态的前提是继承和方法重写，这也是基于抽象类的使用考虑的

二、抽象类的成员特点

  1. 成员变量：既可以是变量也可以是常量（final)

  2. 构造方法：有，用于子类访问父类数据的初始化

  3. 成员方法：既可以是抽象的，也可以使非抽象的

    抽象类里的成员方法不一定都要是抽象的，但只要有一个抽象的方法，这个类就得写成抽象类。而且抽象方法都必须在实例子类里重写以实现多态。非抽象的方法就不需要必须重写了

抽象类的成员方法特性：

抽象方法：强制要求子类做的事（子类必须要重写，所以必须要用）

非抽象方法：子类继承的事，提高代码复用性

三、抽象类中的一些问题

  1. 一个类如果没有抽象方法，可不可以定义为抽象类？如果可以，有什么意义

    可以

    意义是不让创建对象，要访问它只能通过子类

  2. abstract不可以和哪些关键字共存？

    private 冲突

    ​ 加了private，子类就不能对它进行重写，也就失去了强制子类重写的意义

    final 冲突

    static 无意义
